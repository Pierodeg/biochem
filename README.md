# BioChem

A complete management app for a sanitation and analysis services company, handling the day-to-day administration of the business from a single application — available both on mobile and in the browser.

I designed and developed this project entirely on my own as a personal project, from requirements gathering to release.

## Features

- **Client registry** — create, edit and search client records, each with multiple service addresses.
- **Service catalogs** — two dedicated catalogs: laboratory/analysis services and pest control services, with custom price lists.
- **Quotes and invoices** — build quotes by combining clients and services, generate professional PDF documents with in-app preview, and manage invoices.
- **Appointment calendar** — schedule appointments in an integrated calendar, with push notifications delivered to the phone.
- **Parameters register** — keep a register of measured parameters, with bulk import from Excel files.
- **Role-based access** — secure login with user and administrator roles.
- **Cross-platform** — runs on Android and on the web from any browser.

## Tech stack

- **Flutter / Dart** — cross-platform UI (mobile and web)
- **Riverpod** — state management
- **go_router** — navigation
- **Firebase**
  - Authentication — secure login with roles
  - Cloud Firestore — data storage (clients, services, quotes, appointments, register)
  - Cloud Messaging (FCM) — push notifications
  - Cloud Functions — server-side logic (`functions/` folder)
- **pdf / printing** — in-app PDF generation and preview
- **excel** — import of register data from Excel files
- **table_calendar** — calendar UI

## Getting started

1. Clone the repository:
   ```bash
   git clone https://github.com/Pierodeg/biochem.git
   cd biochem
   ```
2. Install dependencies:
   ```bash
   flutter pub get
   ```
3. Configure your own Firebase project (Authentication, Firestore, Cloud Messaging) and replace the configuration generated by the FlutterFire CLI.
4. Run the app:
   ```bash
   flutter run
   ```

## Project structure

- `lib/core/` — theme, routing, shared providers
- `lib/features/` — one folder per feature (auth, anagrafiche, calendario, preventivi, servizi lab, servizi pest, registro, admin, ...)
- `lib/models/` — data models
- `lib/services/` — Firestore services, PDF generation, FCM, Excel import
- `functions/` — Firebase Cloud Functions
